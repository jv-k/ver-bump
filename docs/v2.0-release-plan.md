- Modify: `README.md` (new subsection under *Options*)

**Behaviour:**
- `--github-release` takes no arg; runs `gh release create <TAG> --title <TAG> --notes-file <CHANGELOG-fragment>` after the tag is pushed.
- **Preconditions:** `gh` CLI installed (preflight; else exit `3`); `gh auth status` passes; push target (`-p`) provided.
- **Dry-run:** prints `[dry-run] gh release create ...` without executing.
- **Exit codes:** wire `gh` failure into exit `1` (generic) unless it's an auth/precondition issue → `3`.

**TDD plan (abridged; full steps mirror M3 shape):**

- [ ] Failing test: `--github-release` without `-p` exits 2 ("requires a push target")

- [ ] Failing test: `--github-release` with missing `gh` binary exits 3
- [ ] Implement `check-gh-cli` preflight
- [ ] Implement `do-github-release` (extracts CHANGELOG section for the new tag into a tempfile, invokes `gh release create`)
- [ ] Wire into main flow after `do-push`
- [ ] Add to normalize/parse/usage/completions
- [ ] Docs + PRD update
- [ ] Commit

**Acceptance:**
- Running `VerBump --github-release -p origin -v 1.2.3 -d` in a dirty dogfood repo prints the right `gh` invocation without side effects.
- Non-dry run in sandbox with `gh auth` produces a GitHub release.
